# COMANDOS UTILIZADOS
- git init: Inicializa un nuevo repositorio.
- git status: Comprobamos el estado del repositorio.
- git add .: Añadimos los archivos del repositorio al staging area.
- git commit -m "Estructura de repositorios creada y documentos iniciales añadidos.": Hacemos commit de los cambios del stagging area y con el flag -m añadimos un mensaje personalizado indicando que hicimos.
- git tag -a R1 -m "Primera versión del repositorio de Control de Cambios": Creamos un nuevo tag para el último commit que hemos realizado. Con el flag -a añadimos el nombre del tag, y con el flag -m añadimos una descripción al mismo.
- git branch Mejoras: Creamos la rama Mejoras.
- git checkout Mejoras: Nos movemos a la rama Mejoras.
- git status: Comprobamos el estado del repositorio.
- git add .: Añadimos los archivos del repositorio al staging area.
- git commit -m "Mejoras realizadas por los comentarios obtenidos."
- git checkout master: Volvemos al branch master
- git merge Mejoras: Hacemos merge de la rama Mejoras en la rama master.
- git branch -d Mejoras: Eliminamos el branch Mejoras.
- git add .: Añadimos los archivos del repositorio al staging area.
- git commit -m "Documento de propuestas de mejora eliminado."
- git add .: Añadimos los archivos del repositorio al staging area.
- git commit -m "Corrección de errores en historia.txt": Hacemos commit de la corrección de errores en este archivo (por omisión de dos comandos realizados).
- git tag -a R2 -m "Segunda versión del repositorio de Control de Cambios": Ponemos la etiqueta R2 aportando una descripción de esta versión del repositorio.
- git add .: Añadimos los archivos del repositorio al staging area.
- git commit -m "Nueva versión del documento de proceso de control de cambios."
- git log: Muestra el historial de commits, para localizar el ID del commit al que queremos volver.
- git reset --soft 1a4b1a29f4726611b49c3b359d9971d7ea72f7be: Volvemos al commit anterior manteniendo los archivos.
